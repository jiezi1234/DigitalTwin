# 标准化聊天记录数据模型

## 概述

为了支持多种聊天数据源（微信导出、应用内录制等），项目定义了一套标准化的数据模型，将各种格式的输入数据统一映射到 `ChatRecord` 对象。

## 数据模型定义

### ChatRecord 类

位置：`src/models/chat_record_model.py`

```python
@dataclass
class ChatRecord:
    # ──── 必须字段
    talker: str              # 发送者名称
    msg: str                 # 消息内容
    is_sender: int           # 是否是 self 发送（0=他人, 1=自己）

    # ──── 可选字段（默认值为 None）
    CreateTime: Optional[str] = None      # 创建时间
    MsgSvrID: Optional[str] = None        # 消息服务器 ID
    type_name: Optional[str] = None       # 消息类型
    src: Optional[str] = None             # 消息来源
    room_name: Optional[str] = None       # 所属群组名称
    is_forward: Optional[int] = None      # 是否转发
    id: Optional[int] = None              # 行号或消息 ID
```

## 字段详细说明

| 字段 | 类型 | 必须 | 说明 | 示例 |
|------|------|------|------|------|
| **talker** | string | ✅ | 消息发送者的名称 | `林黛玉` / `self` / `丫头们` |
| **msg** | string | ✅ | 消息内容（不能为空或仅是emoji） | `你好吗？` |
| **is_sender** | int (0/1) | ✅ | 0=他人, 1=自己 | `0` / `1` |
| CreateTime | string | ❌ | 消息创建时间（ISO8601或其他格式） | `2024-01-15 10:30:45` / `1758113387` |
| MsgSvrID | string | ❌ | 微信服务器ID（用于去重和追踪） | `1234567890` |
| type_name | string | ❌ | 消息类型 | `text` / `image` / `voice` / `video` / `link` |
| src | string | ❌ | 消息来源 | `WeChat` |
| room_name | string | ❌ | 所属群组名称（空字符串或null表示私聊） | `红楼梦讨论组` / `""` |
| is_forward | int (0/1) | ❌ | 0=否, 1=是 | `0` / `1` |
| id | int/string | ❌ | 行号或消息唯一标识符 | `1` / `12345` |

## 支持的数据格式

### 格式1：WeChat 标准格式

**必须字段**：`id`, `MsgSvrID`, `type_name`, `is_sender`, `talker`, `msg`, `CreateTime`
**可选字段**：`src`, `room_name`, `is_forward`

**示例 CSV 头**：
```csv
id,MsgSvrID,type_name,is_sender,talker,msg,src,CreateTime,room_name,is_forward
1,12345,text,1,self,你好啊,WeChat,2024-01-15 10:30:45,讨论组,0
```

**来源文件**：`lccc_chat*.csv`, `zhenhuan_chat.csv`

### 格式2：简洁格式

**必须字段**：`is_sender`, `talker`, `msg`
**可选字段**：`id`, `CreateTime`

**示例 CSV 头**：
```csv
id,is_sender,talker,msg,CreateTime
1,0,林如海,汝父年已半百...,0
```

**来源文件**：`daiyu_full_sessions.csv`

## 使用示例

### 1. 从字典创建 ChatRecord

```python
from src.models import ChatRecord

# 来自 CSV 行的数据
row = {
    "talker": "林黛玉",
    "msg": "你好",
    "is_sender": "0",
    "CreateTime": "2024-01-15 10:30:45",
    "room_name": None
}

# 自动验证必须字段，转换类型
record = ChatRecord.from_dict(row)
```

### 2. 自动格式检测

```python
from src.models import ChatRecordSchema

headers = ["id", "MsgSvrID", "type_name", "is_sender", "talker", "msg", "src", "CreateTime", "room_name", "is_forward"]
format_type = ChatRecordSchema.detect_format(headers)
# 返回：ChatRecordSchema.FORMAT_WECHAT_STANDARD

headers = ["id", "is_sender", "talker", "msg", "CreateTime"]
format_type = ChatRecordSchema.detect_format(headers)
# 返回：ChatRecordSchema.FORMAT_SIMPLE
```

### 3. 便捷方法

```python
record = ChatRecord.from_dict(row)

# 检查是否有时间信息
if record.has_time():
    print(f"时间: {record.CreateTime}")

# 检查是否有群组信息
if record.has_room():
    print(f"群组: {record.room_name}")

# 获取消息类型（无type_name时返回 'text'）
msg_type = record.msg_type()
```

## CSV 加载器集成

新的 `WeChatCSVLoader` 使用 ChatRecord 模型：

```python
from src.utils.csv_loader import WeChatCSVLoader

loader = WeChatCSVLoader("csv_clean")

# 自动检测格式、解析数据、转换为 LangChain Document
documents, new_hashes = loader.load(csv_pattern="*.csv")
```

**处理流程**：
1. 检测 CSV 头部 → 判断数据格式
2. 逐行读取 → 使用 `ChatRecord.from_dict()` 解析
3. 过滤无效消息（空、emoji、验证消息等）
4. 转换为 LangChain `Document` 对象
5. 支持增量导入（基于行内容哈希）

## 扩展指南

如需支持新的数据源格式：

1. **识别必须字段**：确保有 `talker`, `msg`, `is_sender` 三个字段
2. **更新检测逻辑**：在 `ChatRecordSchema.detect_format()` 中添加新的格式判断
3. **命名新格式**：如 `FORMAT_CUSTOM_XYZ = "custom_xyz"`
4. **验证映射**：确保 CSV 列名正确映射到 ChatRecord 的字段名

## 错误处理

### 缺少必须字段

```python
try:
    record = ChatRecord.from_dict({"msg": "hello"})  # 缺少 talker
except ValueError as e:
    print(e)  # "Missing required field: talker"
```

### 无法识别的格式

```python
headers = ["col1", "col2", "col3"]
format_type = ChatRecordSchema.detect_format(headers)
# 返回：ChatRecordSchema.FORMAT_UNKNOWN
```

### CSV 加载器中的错误

```python
try:
    docs, hashes = loader.load()
except Exception as e:
    print(f"加载失败: {e}")
    # 日志会显示无效行和处理失败的文件
```

## 性能考虑

- **批量处理**：csv_loader 使用 `csv.DictReader` 流式处理，内存高效
- **并行嵌入**：Document 生成后由 `test_csv_final.py` 进行并行向量嵌入（ThreadPoolExecutor）
- **去重**：基于文件名 + 时间 + 消息内容的哈希（可靠去重 2000+ 万条）
